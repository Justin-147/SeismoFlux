# 阶段3动态异常轨迹特征协议

## 1. 协议身份与阶段边界

- 协议版本：`0.3.0`
- 实施阶段：阶段3“动态异常轨迹特征库”
- 执行模式：`feature_only_no_target_scoring`
- 计划冻结标签：`v0.3.0-anomaly-feature-protocol`
- 冻结日期：`2026-07-15`
- 唯一上位蓝图：`SEISMOFLUX_IMPLEMENTATION_HANDOFF.md`

阶段2R-1的G1-LS已经允许进入阶段3。该许可只允许因果重建异常状态、生成特征、建立来源谱系和完成无评分审计，不允许提前实施阶段4。阶段3不得读取地震标签、目标事件位置、命中结果或成绩，不拟合异常增量模型，不计算信息增益，不运行异常置乱，不判定G2或G3，也不运行锁定测试。

本协议冻结的是“输入到特征”的确定性变换，不宣称异常对未来地震有预测增量。阶段3产出的任何强度、计数、权重或顺位都不是绝对发震概率。

机器可读协议为 `configs/anomaly_history.yaml`。本文与机器配置不一致时，应停止执行并修复协议；不得在执行中自行解释差异。

### 2.1 源列白名单与禁用列

机器配置为两个允许数据集分别冻结源列白名单。任何白名单外源列进入状态或特征即硬失败。

下列语义即使在原始表或标准化元数据中存在，也不得成为阶段3特征：

- 人工预测地点、区域、震级和起止时间；
- 震例、分析意见、预测强度、置信结论和其他人工结果；
- 原表“异常持续时间”；持续时间必须用当时已知的起止状态重新计算；
- `forecast_efficacy`、`term_class`、`R`、`R0`；
- `anomaly_description`等自由文本；
- `annual_anomaly`等未在本协议定义用途的人工分类。

阶段1已经排除了人工预测字段的值；阶段3仍必须用白名单再次阻断，不能仅依赖上游曾经正确处理。

## 3. 起报时刻与因果前缀

阶段3使用 `anomaly_report_period.available_at` 的全部205个不同实际值作为状态快照时刻。首个快照为 `2022-07-20T16:00:00Z`，最后一个快照为 `2026-07-01T16:00:00Z`，分别对应北京时间次日00:00。不得假设固定每周都有报告，也不得人为插入起报日。

对任一快照时刻 `T`，所有处理必须先执行：

```text
eligible observations = observations where available_at <= T
eligible reports      = report periods where available_at <= T
```

过滤必须发生在实体分组、首次出现搜索、结束状态判断、覆盖分母构造、空间聚合和时间窗计算之前。`report_date`、异常真实起始日或后来填写的结束日都不能单独授予可用性。恰好在 `T` 可用的记录纳入，晚于 `T` 一微秒的记录排除。

因此，完整输入和因果前缀在同一 `T` 上必须产生完全相同的科学状态与特征。未来输入可以改变“被排除的未来行数量”等运行审计，但不能改变T时科学载荷。

## 4. 缺报与报告覆盖代理

205个实际报告期之间存在两个已知缺期：

| 年份 | 期号 | 期望报告日 | 冻结处理 |
|---:|---:|---|---|
| 2024 | 36 | 2024-09-04 | `unknown`，不补0 |
| 2025 | 44 | 2025-10-29 | `unknown`，不补0 |

缺整期报告不等于该周没有异常，也不等于上期异常全部消失。缺期不生成合成快照、不填零、不生成“未继续列报”转移。下一实际快照必须记录跨过的缺期数和真实间隔天数。

原始数据没有权威、逐期完整的台站应报名册，因此本阶段只能构造 **reporting coverage proxy（报告覆盖代理）**，不得称为绝对监测覆盖率、台网探测概率或数据完整度。允许的代理量包括：

- 当前实际报告是否存在及其行数；
- 当前有记录的不同台站数、测项数和学科数；
- 距上一实际报告的天数；
- 时间窗内缺少的期数和实际观测快照数；
- 当前报告台站/测项数相对截至T的过去364天因果历史集合之比。

代理分母只能由 `T` 时已经可用的历史台站和测项形成。未来首次出现的台站或测项不得回改历史代理。分母为零时输出null和明确原因，不能输出0。所有覆盖代理特征名必须带 `reporting_coverage_proxy` 语义，公开报告也必须使用同一称谓。

局部报告代理不足只影响对应查询格和尺度，不得使无关区域同时失效。

## 5. 异常状态语义

### 5.1 当前、首次、结束和缺席

每个实际快照至少区分以下状态：

- `current_listed`：完整异常实体出现在当前实际报告；
- `reported_new`：当前行由源表列报为新增；
- `first_seen`：截至T第一次出现的完整异常实体；
- `explicit_end`：截至T已经可用、非右删失且内部一致的明确结束；
- `not_continued`：出现在上一实际报告、未出现在当前实际报告；
- `right_censored`：截至T未得到合格结束证据；
- `left_truncated`：在首个实际报告已经存在，无法知道更早首次出现时间；
- `temporary_entity`：身份不完整、只能在本来源观测内使用的临时实体。

`not_continued`只表示下一份实际报告没有继续列报，不等于异常物理结束。明确结束、报告缺席和缺报必须分列。未来报告的“没有出现”在该报告可用前不能使用。

### 5.2 未来结束、修订与撤回

若某行的结束日在该行知识日期之后，阶段1将其标为未来结束并保持右删失。阶段3必须忽略该结束日；即使后来日历跨过这个日期，也不能自动把早期记录改为已结束。只有后续确认记录在其自身 `available_at` 之后才可改变状态。

结束时间修订或撤回同样只从修订记录可用时向前生效，不得重写早期快照。原表持续时间永久禁用；异常年龄按 `T - start_time` 计算，已知持续时间按合格的 `known_end - start_time` 计算。

### 5.3 左截断和迟入

首份实际报告中的存量异常全部标为左截断，不能计入首次出现率分子。实体第一次在后续报告出现、但起始时间早于前一实际报告时，标记为 `late_entry_or_gap_unknown`，不能武断解释为当期新发生。

### 5.4 重复行和临时实体

- 同一报告内、同一完整 `anomaly_id` 的完全重复行折叠为一个实体状态；原始报告行数代理仍保留源表行数；
- 谱系必须保留全部重复观测ID，不能删除来源证据；
- 同一实体同一报告内的冲突行使用谨慎等级和保守状态；明确结束只有在证据一致时成立；
- `identity_complete=false` 的临时实体不得跨报告链接，不进入纵向首次、斜率、结束或未继续列报特征；
- 临时实体仍可进入原始未加权报告活动代理，并单独报告数量。

## 6. 时间轨迹特征

时间窗固定为4、8、13、26和52周。窗口均为 `(T - 7×周数天, T]`，只使用窗口中真实存在且当时已经可用的快照。

增长斜率按实际经过天数做普通最小二乘，并换算为“每7天”的特征单位；至少需要3个实际快照，否则为null并记录原因。缺期不补零，两个相邻快照相隔14天时，自变量也必须相隔14天。

每个窗口至少保留：

- 当前列报、报告新增、首次出现、明确结束和未继续列报的趋势；
- 实际快照数、缺期期数和真实跨度天数；
- 4周与13周斜率差除以两个窗口中心相隔4.5周形成的增长加速度；
- 当前值相对此前13周实际快照的 `surge_z_13w`；
- 当前值相对截至T历史峰值的回落；
- 异常年龄、已知持续时间和右删失比例。

`surge_z_13w` 的基线只使用 `(T-91天,T)` 内、T以前的实际快照，以有限历史值的算术平均和 `ddof=1` 样本标准差计算：

```text
surge_z_13w = (value_at_T - prior_13w_mean) / prior_13w_sample_std
```

基线至少需要3个实际快照；标准差为0或不可定义时输出null和明确原因。不得把当前值放入自己的基线。所有历史峰值、突增和变化点只能在因果前缀中寻找，不得先在完整时间序列定位后回填到早期快照。

52周峰值使用 `(T-364天,T]` 的因果窗口并包含当前值。`peak_drop_52w = peak_including_current - value_at_T`，因此不得为负；`peak_ratio_52w = value_at_T / peak_including_current`，范围为 `[0,1]`。峰值为0时比例为null并记录原因，不能除以0或填成1。

## 7. 多学科与可靠性特征

多学科特征至少包括不同学科数、不同测项数、学科熵和同一实体的多学科协同比例。自由文本不进入特征。

可靠性固定为三个等级：

| 等级 | 权重 | 用途 |
|---|---:|---|
| `high` | 1.0 | 身份完整且无谨慎/排除标记 |
| `cautious` | 0.5 | 日期错位、重复冲突或结束状态不一致等仍可审计记录 |
| `excluded` | 0.0 | 身份无法解析或缺少纵向实体所需关键字段 |

排除优先于谨慎，谨慎优先于高。空间特征还要求有限经纬度；纵向特征要求完整身份；明确结束要求一致、非右删失的结束证据。

所有主要计数和聚合必须同时保留：

1. 未加权版本；
2. 按1、0.5、0加权的版本；
3. 各可靠性等级的组成计数。

未加权不表示忽略质量：无法形成完整实体的记录只能进入明确标注的原始报告活动代理，不能因为“未加权”而进入实体轨迹。

## 8. 固定查询格和空间尺度

### 8.1 目标无关的25 km格

本地研究区投影到冻结的中国大陆等面积坐标系，以原点 `(0,0)` 铺设25 km固定格。与研究区有正面积交集的格纳入，查询点为精确裁剪交集的内部代表点，格ID只由固定原点的有符号行列号决定。

该网格只由研究区和协议参数决定。禁止使用真实震中、未来地震、命中结果、异常热点或模型输出生成格、细化格或改变边界。阶段3不做12.5 km自适应细化。

固定格、查询点、裁剪几何和研究区边界均为本地受限信息，不进入公开注册表或图件。

### 8.2 闭球与Gaussian核

每个查询点同时生成50、100、200、300和500 km闭球特征，边界规则为 `distance <= radius`。同一快照的闭球成员必须嵌套。

每个闭球尺度同时生成同带宽Gaussian核：

```text
w(d; h) = exp(-0.5 * (d / h)^2),  d <= 3h
w(d; h) = 0,                     d > 3h
h ∈ {50, 100, 200, 300, 500} km
```

Gaussian核分别报告加权和与核权重质量，不能把缺少观测写成真实零强度。

查询域只在本地研究区内，但异常观测不得先按研究区边界裁掉。只要坐标有效，研究区外异常可以按距离影响域内查询格；500 km带宽Gaussian核的最大截断影响距离因此为1500 km。该规则只表示异常空间邻近，不扩大预测研究区。

## 9. 断层字段推迟

虽然总蓝图最终要求断层距离、沿断层投影和断层段聚合，但当前断层快照没有可证明的历史可用时间。阶段3机器配置把全部断层字段固定为 `deferred_to_stage_5`。在阶段5建立长期构造先验和历史可用性协议前，这些字段不能出现在特征字典、Parquet或公开摘要中。

## 10. 来源谱系和随机验收

每个状态行必须记录形成该状态的观测ID、报告ID和最大来源 `available_at`。每个特征行必须记录状态快照ID、特征字典哈希和谱系摘要，并能从 `anomaly_state_history.parquet` 与特征定义确定性重放。

状态表还必须直接保存或无歧义重放 `left_truncated`、`late_entry_or_gap_unknown`、因果 `age_days`、因果 `known_duration_days`、右删失状态、可靠性等级和可靠性权重。仅在文档中描述但无法由状态表重建不算满足验收。

任何悬空来源引用、任何来源晚于起报时刻、任何无字典定义的特征，或任何没有原因的null，均为硬失败。

### 10.1 12个无评分起报日抽查

抽查日期在查看任何阶段4成绩前冻结。开发期和验证期各保留首尾边界，并按命名空间 `stage3-lineage-audit`、种子文本 `147` 的SHA-256顺位选择内部日期：

| 分区 | 抽查日期（北京时间） |
|---|---|
| 开发 | 2022-07-21、2023-05-18、2023-03-16、2022-08-25、2023-04-27、2023-06-29 |
| 验证 | 2024-07-04、2024-10-17、2024-11-07、2025-02-27、2025-04-17、2025-06-26 |

抽查查询格同样只按固定格ID做SHA-256顺位选择，不读取地震或成绩。对选中行检查所有尺度和全部特征：来源可解析、来源可用时间不晚于T、字典定义存在、特征可独立重放、null原因完整。

### 10.2 32个合成因果前缀性质种子

合成测试使用种子0至31，至少验证：

- 完整输入与 `available_at <= T` 前缀产生相同科学载荷；
- 任意改变未来行不影响早期快照；
- 输入顺序不影响身份、值和谱系；
- `available_at == T` 纳入，晚于T排除；
- 五个闭球尺度成员嵌套；
- 一个局部报告代理缺口不屏蔽无关查询格。

这些测试只使用合成异常记录，不读取本地受限Parquet、地震标签或锁定测试数据。

## 11. 输出和公开边界

### 11.1 本地内容寻址输出

```text

├─ anomaly_state_history.parquet
```

`bundle_id`必须共同寻址协议字节、机器配置、环境锁、两个输入数据集的文件/内容/模式哈希、研究区哈希、代码提交和确定性序列化规则。相同ID的不同字节为硬失败；本地输出不得提交到公开Git仓库。

- `data/manifests/anomaly_feature_registry.json`
- `data/manifests/anomaly_feature_dictionary.json`

公开审计SVG只能显示时间覆盖、缺报、可靠性组成和审计通过率等聚合图，不得绘制受限边界或空间点位。

## 12. 锁定测试与阶段4停止线

阶段3锁定测试证明必须始终为：

```text
run = false
issue_dates = null/null
target_count = null
target_ids = []
score_ids = []
artifact_ids = []
result = null
action = do_not_run
```

阶段3不得创建锁定测试日期，不得读取锁定目标，不得把正式锁定评估写成普通测试命令。

在阶段3完成测试、验收、提交和推送前，阶段4必须停止。即使阶段3通过，也只能随后按独立预登记进入可解释异常增量模型；地震标签、评分、异常时间/空间置乱和G2/G3判定全部属于阶段4，断层字段属于阶段5。

## 13. 阶段3验收门槛

只有同时满足下列条件才允许进入阶段4：

1. 205个实际报告期全部因果重建，快照数不多不少；
2. 任一状态或特征来源的 `available_at` 均不晚于对应T；

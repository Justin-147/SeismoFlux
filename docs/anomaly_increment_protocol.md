## 3. 冻结科学问题：H1与H2

### 3.1 H1：异常是否提供增量信息

- 假设：动态异常在最佳可评估地震背景之上提供增量信息。
- 确认性对照：`background_no_increment`。
- 报告覆盖混杂对照：`coverage_only`。
- 必须保留的安慰剂：1,000次时间置乱和1,000次空间置乱。
- 通过含义：动态模型相对背景的配对信息增益为正、超过时间置乱、超过覆盖控制，并在冻结报警面积规则下达到预登记的实际改善。
- 不通过含义：当前数据和模型下未发现可信异常增量；发布负结果并停止复杂异常模型。

### 3.2 H2：多期轨迹是否优于单期快照

- 假设：因果多期轨迹稳定优于只使用当前一期异常的快照模型。
- 对照：相同背景、覆盖控制、预处理、惩罚、拟合曝光和评估曝光下的 `snapshot`。
- 通过含义：`dynamic - snapshot` 在三个互斥目标带滚动折中至少两个折为正，且三折宏平均差的中位数为正。
- 不通过含义：不声称动态演化有效，并按预登记采用矩阵保留快照或背景。

任何假设未通过时保留其对照，不以模型复杂度作为采用理由。只有G2与G3支持后才允许讨论后续构造先验；神经模型仍需更晚门控。

## 6. 预处理、共享震级率头与点过程目标函数

### 6.1 精确预处理顺序

所有统计量只来自当前折7天拟合曝光，或正式验证前的全开发7天拟合曝光。验证和评估值不能改变预处理器。统计行按“拟合起报日 × 正面积25 km格”不加权等权计算。

精确顺序为：

1. 验证源值域，只用有限值估计统计量；
2. 按逻辑特征应用冻结变换；
3. 用NumPy `linear`分位算法计算0.5%和99.5%训练分位数；
4. 裁剪连续变换值；
5. 在变换空间用训练中位数填补缺失；
6. 连续尺度依次尝试 `1.4826 × MAD`、`IQR / 1.349`、训练总体标准差；
7. 用训练中位数中心化并按所选尺度标准化；
8. 追加不裁剪、不标准化的二元缺失指示列。

变换分组冻结为：

- `log1p_nonnegative`：计数、报告间隔、缺期数、年龄、扩散半径；
- `asinh_signed`：4周/13周斜率、4对13周加速度、13周突增和52周峰值回落；
- `identity_binary`：当前报告是否存在；
- `identity_finite`：覆盖比、多学科比例、归一熵和集中度。

null不等于0。每个值列都生成缺失指示；有效性、样本数和null原因等质量伴随列只保留审计，不得进入模型。某列在拟合范围没有有限值时，变换值和系数固定为0，仍从原始null位图生成缺失指示；评估/验证数据不得救活该列。训练最小值等于最大值时才把值系数固定为0。所有预处理统计、回退分支和列身份都必须序列化并哈希。

### 6.2 共享震级率头

M5-6与M6+共享同一异常系数向量，但各有一个Poisson对数率截距。两个率截距只在当前共享7天拟合范围、使用无增量背景拟合一次，然后由背景、coverage、snapshot、dynamic和全部安慰剂原样复用；评估或验证不得重拟合。

- M5-6拟合范围零事件：该范围及相应门控直接记为证据不足，不启动优化器；
- M6+拟合范围零事件：该震级档强度精确置0并从联合目标排除，只报探索证据不足；
- 禁止负无穷数值截距、伪计数或看分后先验。

### 6.3 目标函数与提前期衰减

对每个变体、每个拟合范围，最小化未按行数或事件数归一的目标：

```text
J(beta) = Σ integrated_intensity
        - Σ event_log_intensity
        + 0.5 * ridge_lambda * Σ penalty_factor_j * beta_j^2
```

冻结背景对数强度、格面积和时间曝光是offset。异常增量不另设截距；震级率截距不惩罚，覆盖项、异常项和缺失指示的惩罚因子均为1。优化器为SciPy L-BFGS-B float64，确定性零初始化，`maxiter=2000`、`maxfun=100000`、`ftol=1e-12`、`gtol=1e-7`；不收敛时登记失败，不能放宽阈值。

起报T后提前期u的增量线性预测子为：

```text
eta_variant(T,u) = 2^(-u/90) * X_variant(T) · beta_variant
```

覆盖、快照和轨迹增量项一起衰减；冻结背景和震级率头不衰减。窗口内禁止读取未来异常或覆盖更新。无增量路径必须数值上精确退化为“共享震级率头 × 冻结背景”。

## 7. 空间积分、时间积分与事件项

### 7.1 补偿积分

- 等面积主网格为25 km，收敛网格为50、25和12.5 km，原点和范围沿用阶段2冻结契约；
- 每个网格中心都从已接受的异常状态历史重算同一200 km快照和轨迹特征，禁止从25 km特征库插值或分片常数复用；
- 跨网格保持同一预处理器和系数，不得重拟合；
- 精确裁剪面积作为权重，目标或震中不得参与建格和加密；
- 25 km相对12.5 km总强度差必须不超过0.5%；50 km只报粗尺度趋势；近零绝对容差为`1e-10`；
- 时间积分在开闭区间 `(T,T+h]` 使用复合中点法，主步长1天、参考步长0.5天，最后不足一步使用自身中点；相对差必须不超过0.5%，近零绝对容差同为`1e-10`。

空间或时间积分不收敛即判模型失败，并禁止发布空间结果。

### 7.2 事件对数强度项

事件只在评分时映射到冻结25 km格：

1. 用等面积格多边形的`covers`判断所属格；
2. 边界并列按行、列、`cell_id`升序；
3. 使用该格中心的强度，不在精确震中重算特征；
4. 异常衰减使用事件时刻减起报时刻的精确提前期；
5. 50/25/12.5 km补偿收敛审计时，事件项始终保持上述25 km定义不变。

目标事件不得创建或细化格、改变候选、区域或边界。信息增益使用包含积分补偿的点过程对数似然差，并按受支持的唯一物理事件数报告nats/事件。

### 10.1 构造分层只用于置乱与地区诊断

拓扑算法固定为：WGS84投影至冻结Albers并裁剪到本地研究区；内部开放端点确定性连接到最近其他线段或研究区边界，最大100 km；先做exact unary-union noding，再固定到1 m精度，最后`polygonize_full`。

拓扑门要求cuts、dangles和invalid均为0；0.001、0.01、0.1、1和10 m精度下分区数一致；输入线段顺序改变不改变身份；每个固定25 km查询格唯一归区。冻结结果为65个区、39个有查询格区和26个零查询格区，15,697个查询格全部唯一赋区。

评分前还必须同时核对五个冻结输入：L1线划、L2线划、阶段3特征库、阶段3异常状态历史和本地研究区。公开空间清单中的`spatial_placebo_implementation_authorized=true`只授权无评分空间安慰剂实现；`stage4_target_read_authorized=false`明确禁止读取目标。前者不替代第2节的评分代码标签与执行封印，不能单独授权评分。

### 10.2 实体坐标置换

空间零假设从已接受的阶段3`anomaly_state_history`实体行出发，不交换聚合格分数：

- 分组单元是“起报期 × 构造分层”；组内对完整经纬度坐标对做双射；
- 异常身份、可靠性、学科/测项、年龄/状态及其他非坐标科学属性固定；
- 收件和供件行按稳定`state_id`升序，重复坐标也按该顺序确定配对；
- 每个组保持坐标对多重集和实体数不变；
- 空间不可用实体固定排除，永不进入置换池；
- 研究区外实体使用“最近构造区ID × outside标志”独立分层，绝不与区内实体交换；
- 零查询格区仍使用本地受限区几何和实体坐标独立置换，禁止跨区兜底；
- 单例组或全相同坐标组执行恒等映射并报告无效置乱比例；
- 置换后先重算选定200 km快照科学场，再从各期置换场重算4/13/52周轨迹；覆盖代理从原始接受值精确回接；
- 同一坐标映射跨模型变体、震级档和五个窗口共享；每次都重拟合预处理器和异常系数模型，但共享震级率头必须与真实模型相同并原样复用。

每次重复记录映射SHA-256、移动实体行数、固定点数、单例层数、无效层比例和每期每区坐标多重集核验。该零假设不检验未标记的台站密度本身，解释时必须保留这一限制。

## 11. 带类型、跨维度配对的随机流

根种子固定为147，生成器固定为`numpy.random.PCG64`。调用方不得传自由命名空间字符串或用`root_seed + index`派生子种子。

每个随机上下文必须使用冻结类型：

- 公共字段：整数`schema_version=1`、字符串`protocol_version=0.4.0`、64位小写十六进制输入封印、枚举`purpose`、冻结`evaluation_id`、合法`partition_role`、冻结`pairing_scope`和非负整数`replication_index`；
- `evaluation_id`只允许`development-fold-1/2/3`和`formal-validation`；
- Bootstrap的`partition_role`只能为`joint`；时间/空间置乱只能为`fit`或`assessment`；
- 空间置乱额外要求非空字符串`issue_id`与`construction_stratum_id`；其他随机族禁止这两个字段。

上下文以UTF-8、键排序、无NaN的紧凑canonical JSON序列化，计算：

```text
SHA256("seismoflux-stage4" NUL "147" NUL canonical_context_json)
```

取前16字节按big-endian解释为PCG64熵。冻结输入封印绑定规范化协议、折清单、特征清单、含本地工件哈希的空间清单、背景身份、阶段3身份和预期评分代码标签。

`model_variant`、`magnitude_bin`、`horizon`、`metric`和`worker_count`明确禁止进入种子上下文；它们可以进入结果身份，但必须共享同一个随机映射/事件权重。随机流因此跨模型变体、震级档、五个窗口和指标配对。输出按重复索引升序汇集，改变工作进程数不得改变结果。四个评价范围的第0、1和最后一次重复都必须匹配 [data/manifests/anomaly_increment_randomness.json](../data/manifests/anomaly_increment_randomness.json) 中的独立参考向量。

## 12. 事件簇Bootstrap、置乱失败与多重比较

### 12.1 五窗membership × 震级分层Bootstrap

Bootstrap固定2,000次、95% percentile区间，不重新拟合模型：

1. 对每个唯一物理事件形成按7、30、90、180、365天顺序的五位非空窗口membership签名；
2. 按“震级档 × 五位签名”分层；
3. 每层按原事件数有放回抽样，单例层确定性自抽，空层不创建；
4. 同一事件权重跨变体、五个窗口和所有指标共享；
5. 因分层抽样，每个窗口的边际事件数保持不变；
6. 积分补偿保持预登记曝光的固定值；信息增益分母保持原支持域唯一事件数，召回分母保持原全研究区唯一事件数；
7. 同一重复同时重算信息增益、同面积召回和同召回面积。

G2只取M5-6与7/30/90天分量；M6+或长窗口成员不得替换主成员。任一主窗口观测事件为0时，G2记为证据不足。

### 12.2 置乱重复失败

时间、空间各1,000次，每次必须使用原映射重新拟合预处理器和异常系数模型，同时原样复用冻结共享震级率头。科学不收敛或统计量非有限时，将该零统计量记为正无穷并计入“null ≥ observed”；基础设施中断只能从已验证检查点恢复同一映射，禁止丢弃或替换。科学失败比例超过1%则证据不足。Monte Carlo分母始终为1,001：

```text
p = (1 + count(null >= observed)) / 1001
```

时间置乱的M5-6三主窗口等权宏平均信息增益是G2唯一确认性置乱端点，不做多重校正。窗口×震级探索检验使用Holm族错误率校正，时间和空间各自成族。

## 13. G2：正式验证上的精确判定

G2候选固定为`dynamic`，对照固定为`background_no_increment`，只在一次正式验证的M5-6、7/30/90天上判定，三个窗口等权。必须全部满足：

1. 三个窗口都可评估，且每个窗口至少有1个评分事件；
2. 三窗口物理事件ID去重并集至少20个；
3. 三个窗口相对背景的信息增益点估计都为正；
4. 三窗口等权宏平均信息增益的95%区间下界大于0；
5. 1,000次时间置乱的主端点`p <= 0.05`；
6. `dynamic - coverage_only`三窗口等权宏平均信息增益的95%区间下界大于0；
7. 以下实际改善至少一项通过。

### 13.1 同面积严格召回分支（`same_area_strict_recall_gain_percentage_points`）

- 面积预算固定600,000 km²；
- 每个起报期按“条件期望强度降序、行、列、cell_id升序”选择不超过预算的最大完整25 km格前缀；
- 不裁切最后一格，不使用真实震中改变顺序；
- 全研究区严格召回中无支持区目标计为漏报；
- 各窗口先计算`dynamic - background`召回差，再对7/30/90天等权宏平均；
- 宏平均提高至少5个百分点，且95%区间下界大于0。

### 13.2 同召回面积分支（`same_recall_union_area_relative_reduction`）

该分支不做连续插值，算法固定为：

1. 对每个主窗口，先取背景在600,000 km²下的严格命中数作为参考；
2. dynamic只在0至960,000 km²、步长625 km²的含端点预算格上搜索；
3. 每个预算仍选择不超过预算的最大完整确定性格前缀；
4. 取达到背景参考命中数的最小预算，并使用实际选中完整格前缀的精确面积，而不是名义预算；
5. 在该窗口的非重叠曝光间等权平均精确面积；
6. 每窗口下降率为`1 - dynamic_mean_area / background_mean_area`，再对三个窗口等权平均；
7. 宏平均面积下降至少10%，且95%区间下界大于0。

若背景参考命中数为0，或dynamic在960,000 km²仍达不到参考命中数，该分支不可评估且不能通过。Bootstrap中某次该分支失效时保留数值下降0作为失败，禁止删除该重复。Bootstrap只改变事件权重、命中数和由此选择的预算，冻结面积值本身不重算。

G2任一科学条件失败，发布可信负结果并停止复杂异常模型；样本或数值条件不足则发布“证据不足”，不得放宽阈值。

## 14. G3与预登记采用矩阵

G3只使用三个开发滚动折，不使用正式验证反向判断动态特征：

- 每折计算7、30、90天的`dynamic - snapshot`信息增益（nats/物理事件）；
- 三个窗口等权形成折统计量；
- 任一折任一窗口零评分事件，则G3为证据不足，不计算部分宏平均；
- 三个互斥目标带必须全部存在；
- 至少两个折统计量为正，且三折统计量中位数大于0；
- 每折每变体只拟合一个共享7天模型，三个窗口共用同一参数向量。

采用矩阵在看分前固定：

| 结果 | 采用决定 |
|---|---|
| dynamic通过G2且通过G3 | 允许dynamic进入阶段5比较 |
| dynamic通过G2、G3失败，且snapshot按同一G2阈值/置乱/实际指标通过 | 只采用snapshot |
| dynamic通过G2、G3失败，snapshot等价G2也未通过 | 回到background_only |
| dynamic未通过G2 | 回到background_only并停止复杂异常模型 |
| coverage_only | 只作报告覆盖混杂诊断，永不作为异常采用 |

禁止看分后发明回退规则。只在单一地区改善不能声称稳定增量；必须按构造区和窗口报告信息增益、独立事件数与严格召回。

## 15. CPU权威路径与GPU等价门

权威路径为CPU float64：预期24个物理核心，至少保留2个，最多12个工作进程；禁止嵌套并行，每个进程BLAS线程固定1。每25次置乱写检查点，每30秒心跳，并记录CPU、内存、GPU、耗时和并行设置。

GPU当前只能视为候选加速器，必须在第二冻结门以前通过CPU等价测试：

- 随机映射必须在CPU生成并原样传入GPU；
- GPU必须使用float64；
- 目标函数相对误差`<=1e-10`；
- 梯度最大绝对误差`<=1e-8`；
- 系数最大绝对误差`<=1e-8`；
- 积分强度相对误差`<=1e-8`；
- 科学输出在上述容差内身份一致，且GPU不得改变特征、超参数、随机流或失败处理。

若评分代码冻结时GPU后端未安装或任一等价测试失败，正式运行锁定为CPU float64；不得在看分后启用GPU，也不得放宽容差。GPU可用不构成采用深度模型的理由。
